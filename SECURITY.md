# Security Policy

## Supported Versions

Security fixes are provided for the latest release only.

| Version | Supported |
| ------- | --------- |
| 8.x (latest) | Yes |
| 7.x and earlier | No |

## Reporting a Vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Report security issues privately through [GitHub Security Advisories](https://github.com/ne0lith/tiktok-live-recorder/security/advisories/new).

Include as much detail as possible:

- Affected version
- Steps to reproduce
- Impact assessment
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** within 7 days
- **Initial assessment:** within 14 days
- **Fix or mitigation:** depends on severity; critical issues are prioritized

## Out of Scope

The following are **not** security vulnerabilities for this project:

- TikTok account bans, rate limits, or Terms of Service violations
- Stream access blocked by regional restrictions or WAF changes
- Cookie or session expiry
- Missing or invalid FFmpeg installation
- User misconfiguration of `config/*.json`

## Code of Conduct Reports

Code of Conduct violations should also be reported privately — not in public issues. Use [GitHub Security Advisories](https://github.com/ne0lith/tiktok-live-recorder/security/advisories/new) and note that the report is a Code of Conduct matter.
